## Challenges and Growth Areas

1. **`_bmad-enhanced/` vs `bmad-enhanced` substring collision** — The sed pipeline cannot distinguish between `bmad-enhanced` as a product name and `_bmad-enhanced/` as a historical directory name — both contain the target substring. The mitigation (targeted post-replace revert) worked, but the root cause is inherent to context-blind replacement tools.

2. **Verification grep commands semantically inverted by sed** — `epics-phase3.md` contained grep commands that searched for OLD stale product names. Sed replaced those old names with new names, changing "search for stale refs" into "search for current names" — semantically meaningless. Caught by code review (M1), not by grep verification.

3. **`BMAD Enhanced` (space-separated) audit inaccuracy** — Initial story audit said 0 instances would appear, but 8 instances were found in 6 root-level files during Task 1 execution. The pipeline was expanded mid-story. Previous stories (3.2) also found 0 instances, creating false confidence.

---

## Key Insights

1. **`git diff` review catches a different class of errors than grep verification** — Grep confirms pattern presence/absence. `git diff` confirms correctness of replacements in context. Both are needed for bulk replacement operations. This should be a standing practice for any future sed/replace operations touching >10 files.

2. **Context-blind replacements require known false positive categories** — The `_bmad-enhanced/` directory name collision was predictable in hindsight — any file referencing historical directory structures would hit this. Maintaining a list of known false positive categories (like `_bmad-enhanced/` paths) and running targeted revert passes is the correct mitigation.

3. **Files containing verification/search commands need separate audit** — When sed-replacing in files that contain grep/search commands, those commands must be audited separately to ensure their search TARGETS aren't corrupted. The search command itself is the "meta" level — it's supposed to reference the OLD patterns.
