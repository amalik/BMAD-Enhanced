# v3.2.0 — Portability, Team Factory, and Test Recovery

Export your BMAD skills to any AI platform, ship the Team Factory, and build on a test suite you can actually trust.

## Highlights

### Portable Skills

BMAD skills are no longer locked to Claude Code. The new portability system exports any skill to a standalone, LLM-agnostic format with ready-to-use adapters:

```bash
npx convoke-export bmad-brainstorming --output ./exported
```

- **Platform adapters** for Claude (`CLAUDE.md`), GitHub Copilot (`.github/copilot-instructions.md`), and Cursor (`.cursor/rules/`)
- **Tier classification** — standalone skills export cleanly; pipeline skills that depend on multi-step orchestration are flagged
- **Export validation** catches structural issues and BMAD-internal leaks before you ship
- **Catalog generator** creates a browsable skill catalog README from your manifest

New skills: `bmad-export-skill`, `bmad-generate-catalog`, `bmad-seed-catalog`, `bmad-validate-exports`

### Team Factory Ships

The Team Factory module (`_bmad/bme/_team-factory/`) is now included in the npm package. Create fully-wired, BMAD-compliant agent teams through a guided workflow — composition patterns, overlap detection, contract design, and integration wiring included.

### Test Infrastructure Recovery

The v3.1.0 test suite had a hidden problem: ~4,400 lines of tests existed on disk but never ran (written in Jest API for a `node:test` project). This release resolves that completely:

- **22 test files** converted from Jest to `node:test`
- **Test gate: 1,123 tests** (was 320+ reported, ~408 actually running)
- **Structural defenses** installed: eslint rejects undeclared Jest globals, `npm test` glob auto-discovers new test files, Definition of Done requires runtime test-count verification
